# Plan: Routines (2026-07-24)

Implements [ADR 0011](../adr/0011-routines.md). Center of gravity = **inbox + notification**, execution is thin. Vertical slices; each slice ships a working thread end-to-end.

## Slice 0 — Skeleton loop (walking skeleton)

Prove the whole loop with the smallest surface: one hardcoded/CRUD-less routine → ticker → `Engine.Create` → run completes.

- **Store**: migration adding `routines` table + `tasks.routine_id` (nullable). Model in `internal/store`.
- **Scheduler**: `internal/routines/scheduler.go` — single ticker goroutine modelled on `internal/task/approvals.go` `StartExpiryLoop(ctx, interval)`. On tick, select `enabled && next_due_at ≤ now`, dispatch via `Engine.Create` with routine fields + `routine_id`, then `next_due_at += interval_secs` (bounded catch-up) + per-routine jitter.
- **Mount**: start the ticker in `internal/server/server.go` right where `StartExpiryLoop` is wired.
- **Accept**: a seeded routine fires on schedule, creates a Task tagged `routine_id`, and a normal interactive task is unaffected.

## Slice 1 — CRUD + API

- **Store**: routine CRUD (`Create/List/Get/Update/Delete`, `SetEnabled`).
- **API**: `internal/server` routes `GET/POST /api/routines`, `GET/PATCH/DELETE /api/routines/{id}`, `POST /api/routines/{id}/run-now` (manual trigger → same `Engine.Create` path; ignores schedule).
- **Accept**: create/pause/delete/run-now via HTTP; `go test ./internal/routines/... ./internal/store/...` green.

## Slice 2 — Report signal + notification (the actual product)

- **Prompt convention**: routine prompt template instructs the run to end with a one-line TL;DR + `noteworthy: true|false`. Parse the signal from the run result/artifact.
- **On completion hook**: always append to runs feed; push via existing notify (Bark/ntfy) **only if `noteworthy`**. Reuse `notify` + `ui.base_url` deep-link (settings already expose these).
- **Resilience**: record errors on the run; `consec_failures` circuit breaker auto-disables after N and pushes one alert.
- **Accept**: "0 new PRs" run → silent into feed; "PR touched auth middleware" → phone buzzes with TL;DR + deep link. A run panic/timeout records `failed`, never blocks interactive tasks.

## Slice 3 — UI: read/write separation

- **Global Routines tab (read + alert)**: new `Route path="/routines"` in `ui/src/App.tsx` + `NavLink` in `ui/src/components/layout/Sidebar.tsx` footer (parallel to `/agents`); add `IconRoutines`. Unread pill mirroring the Inbox pending-badge (`App.tsx` count + WS `task_update` where `routine_id` set). Page = reverse-chron runs feed reusing `ApprovalsPage` card-list + `subscribeWS` pattern; noteworthy pinned, silent runs folded. Per-routine enable/disable/delete/run-now.
- **Per-project button (write + context)**: button in `ui/src/pages/ProjectDetailPage.tsx` header action row (beside *Continue focus*) → create routine pre-scoped to project cwd; show "N routines on this project."
- **i18n**: add `nav.routines` + a `routines: {...}` namespace to both `ui/src/i18n/locales/en.ts` and `zh.ts`.
- **Build**: `npm run build`; commit rebuilt `web/dist`.
- **Accept**: create from a project, watch a run land in the global feed with unread badge, get the push, mark read.

## Explicitly not doing (hold the "small")

Probe / command templates · cron-expression editor · DAG / conditional branches / workflow engine · per-routine priority queue · routine-triggers-routine · system cron/launchd dispatch. (See ADR 0011 Non-goals.)

## Ship checklist

1. Migration idempotent; `routines` + `tasks.routine_id` present.
2. Ticker catch-up + jitter; restart does not stampede.
3. run-now + scheduled both go through `Engine.Create` on the shared FIFO queue.
4. Timeout/error → `failed` + circuit breaker + one alert; **interactive tasks unaffected**.
5. Report signal parsed; noteworthy→push, else silent-feed.
6. `go test ./internal/routines/... ./internal/store/...` green; `npm run build` passes; `web/dist` committed; en+zh both updated.
